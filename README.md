## 为什么需要这个？

你对着空白文档发呆的时候，缺的不是文笔，是框架。

"帮我写个数字化转型方案"——你从哪开始写？背景？目标？举措？还是先找个模板抄？

一句话出方案解决的就是这个"从0到1"的问题。你丢一句话进来，我帮你把框架搭好，论点填上，逻辑检查一遍。剩下润色的事，你自己来。

> 方案最难的不是写，是开始写。

## 怎么用

```
你："帮我写个政务数字化方案"
我："收到。这个方案给谁看？领导还是技术团队？"
你："给局长看的汇报材料"
我：[3分钟输出完整框架]
```

就这么简单。一句话进，框架出来。

## 5步闭环

```
理解需求 → 定位受众 → 搭框架 → 填论点 → 检查逻辑
```

| 步骤 | 做什么 | 为什么 |
|------|--------|--------|
| 1. 理解需求 | 确认核心问题、受众、场景 | 方向对了再走 |
| 2. 定位受众 | 决定语气和深度 | 给领导写和给同事写不一样 |
| 3. 搭框架 | 背景+目标+举措+保障+成效 | 骨架有了肉才好填 |
| 4. 填论点 | 每个举措用"动作+对象+效果" | ❌"加强培训" ✅"面向全区窗口人员，开展一网通办实操培训，考核通过率提升至95%" |
| 5. 检查逻辑 | 自检5问 | 目标能衡量吗？举措和目标对得上吗？领导看完会问"然后呢"吗？ |

## 三大场景模板

### 🏛️ 政务方案
```
背景与依据 → 总体目标 → 重点任务 → 保障措施 → 预期成效
```
政策背景+量化指标+考核机制，局长看了直点头。

### 💼 商业方案
```
项目背景 → 核心目标 → 核心策略 → 执行计划 → 资源需求 → 成功指标
```
SMART目标+甘特图视角+北极星指标，投资人看得懂。

### 💻 技术方案
```
需求分析 → 技术选型 → 建设内容 → 实施路径 → 风险与对策 → 资源评估
```
架构图+选型理由+成本估算，CTO没话可说。

## 输出示例

**你说的**："给我一个关于推广智能客服的方案"

**我出的**：

> **XX单位智能客服推广方案**
> 
> **一、背景**：根据国务院指导意见，群众对7×24小时服务需求迫切，我单位当前热线人工接听率仅65%
> 
> **二、目标**：智能客服独立解决率达70%，人工接听率降至40%，等待时长压缩至30秒
> 
> **三、重点任务**：
> 1. 面向高频咨询场景，搭建智能问答知识库（覆盖80%常见问题）
> 2. 面向全区窗口人员，开展智能客服实操培训（考核通过率95%）
> 3. 面向群众端，上线多渠道智能入口（微信/APP/网站三端覆盖）
> 
> **四、保障**：成立专项工作组，预算XX万，按月考核进度
> 
> **五、成效**：直接降低人力成本30%，间接提升群众满意度15个百分点
